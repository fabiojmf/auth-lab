Glossário

Referência rápida dos termos usados ao longo do treinamento.

Access Token
Token JWT enviado nas requisições a APIs protegidas. Contém permissões (scopes/roles) e tem vida curta (geralmente 5 min). Trilha 01, módulo 03.
Authorization Code
Código temporário (de uso único) retornado pelo Keycloak após login bem-sucedido. É trocado por tokens no backend. Trilha 03, módulo 01.
Claim
Um campo dentro de um token JWT. Cada claim é uma "afirmação" sobre o usuário (ex: email, name, roles). Trilha 01, módulo 04.
Client
Uma aplicação registrada no Keycloak que pode solicitar autenticação. Pode ser confidential (com secret) ou public (sem secret). Trilha 04.
Client Credentials
Fluxo OAuth2 para comunicação máquina-a-máquina (sem usuário humano). O client se autentica com seu próprio ID e secret. Trilha 03, módulo 02.
Confidential Client
Client que pode guardar um segredo (secret) de forma segura — tipicamente aplicações com backend. Trilha 04, módulo 01.
CORS (Cross-Origin Resource Sharing)
Mecanismo do browser que controla quais origens podem fazer requisições a uma API. Configurado no Keycloak via Web Origins. Trilha 06, módulo 03.
Direct Grant (Resource Owner Password)
Fluxo onde a aplicação recebe diretamente usuário/senha e troca por token. Uso restrito a cenários legados. Trilha 03, módulo 03.
Grant Type
O "tipo de concessão" usado para obter tokens. Exemplos: authorization_code, client_credentials, refresh_token.
ID Token
Token JWT emitido pelo OIDC que contém informações de identidade do usuário (quem ele é). Não deve ser enviado a APIs. Trilha 01, módulo 03.
Identity Provider (IdP)
Sistema que autentica usuários e emite tokens. O Keycloak é um IdP. Google, GitHub e Azure AD também são IdPs. Trilha 05, módulo 03.
Issuer (iss)
Claim no JWT que identifica quem emitiu o token. Formato: http://localhost:8080/realms/authlab. Usado para validação.
JWT (JSON Web Token)
Formato de token compacto com 3 partes: header.payload.signature. Codificado em Base64URL e assinado digitalmente. Trilha 01, módulo 04.
OAuth 2.0
Framework de autorização que permite a aplicações obter acesso limitado a recursos em nome de um usuário, sem expor suas credenciais. Trilha 01, módulo 03.
OIDC (OpenID Connect)
Camada de identidade construída sobre o OAuth2. Adiciona o ID Token e o endpoint /userinfo. Trilha 01, módulo 03.
PKCE (Proof Key for Code Exchange)
Extensão de segurança para o Authorization Code Flow. Usa code_verifier + code_challenge para proteger a troca do code. Obrigatório para public clients. Trilha 03, módulo 01.
Public Client
Client que NÃO pode guardar um segredo — SPAs (React, Angular), apps mobile. Usa PKCE em vez de client_secret. Trilha 04, módulo 01.
Realm
Espaço isolado no Keycloak com seus próprios usuários, clients, roles e configurações. Equivalente a um "tenant". Trilha 02, módulo 02.
Redirect URI
URL para onde o Keycloak redireciona o browser após login. Deve ser pré-registrada no client para segurança.
Refresh Token
Token de longa duração usado para obter novos access tokens sem re-autenticar o usuário. Trilha 03, módulo 04.
Role
Papel atribuído a um usuário que define suas permissões. Pode ser realm role (global) ou client role (específico de um client).
Scope
Define quais informações/permissões o client está solicitando. Exemplos: openid, profile, email. Trilha 04, módulo 03.
Service Account
Conta associada a um client (não a um humano) usada no fluxo Client Credentials. Trilha 04, módulo 04.
SSO (Single Sign-On)
Mecanismo que permite ao usuário fazer login uma vez e acessar múltiplos sistemas sem re-autenticar. Trilha 01, módulo 02.
State
Parâmetro aleatório enviado no Authorization Request e verificado no callback. Protege contra ataques CSRF.
Token Exchange
Mecanismo para trocar um token por outro com escopo diferente — delegação, impersonation ou downscoping. Trilha 09, módulo 06.
User Federation
Integração do Keycloak com fontes externas de usuários (LDAP, Active Directory). Trilha 05, módulo 02.
Well-Known (Discovery Endpoint)
Endpoint /.well-known/openid-configuration que expõe todos os endpoints e capacidades do servidor OIDC.
