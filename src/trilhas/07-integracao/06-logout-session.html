Logout e Session Management

O logout parece simples, mas em SSO é um dos problemas mais complexos: o usuário faz logout na sua app, mas continua logado no Keycloak (e em todas as outras apps).

🚪

O problema do logout em SSO

Limpar a sessão local não é suficiente. Logout completo exige coordenação entre sua app E o Keycloak.

Tipos de Logout

Tipo O que faz Quando usar
Application-only Limpa sessão local. Usuário continua logado no KC. Quase nunca — confunde o usuário
RP-Initiated Redireciona ao Keycloak, encerra sessão SSO Sempre — é o logout padrão
Backchannel Keycloak notifica sua app quando logout ocorre em outro lugar Produção — para logout coordenado entre apps

RP-Initiated Logout

O usuário clica "Logout" na sua app / você redireciona para o endpoint de logout do Keycloak:

GET /realms/meu-realm/protocol/openid-connect/logout?
  id_token_hint=eyJhbGciOiJSUzI1NiIs...
  &client_id=minha-app
  &post_logout_redirect_uri=https://minha-app.com/
Parâmetro Obrigatório Função
id_token_hint Recomendado Prova qual usuário está fazendo logout (evita tela de confirmação)
client_id Sim (se sem id_token_hint) Identifica o client. Obrigatório no Keycloak 18+ quando id_token_hint está ausente.
post_logout_redirect_uri Opcional Para onde redirecionar após logout (deve estar configurado no client)

Sem id_token_hint, o Keycloak mostra uma página de confirmação pedindo ao usuário para confirmar o logout.

Backchannel Logout

Quando o usuário faz logout em outra aplicação, o Keycloak precisa notificar a sua. O Backchannel Logout faz isso via POST server-to-server:

Fluxo:
1. Usuário faz logout no App A
2. App A redireciona ao Keycloak (RP-Initiated)
3. Keycloak encerra sessão SSO
4. Keycloak envia POST para TODAS as apps com backchannel configurado
5. Cada app recebe um logout_token e invalida a sessão local

POST https://minha-app.com/backchannel-logout
Content-Type: application/x-www-form-urlencoded

logout_token=eyJhbGciOiJSUzI1NiIs...

O logout_token é um JWT contendo:

{
  "iss": "https://keycloak/realms/meu-realm",
  "sub": "f1234-abcd-5678",
  "sid": "session-id-do-keycloak",
  "events": {
    "http://schemas.openid.net/event/backchannel-logout": {}
  }
}

Sua app deve: validar o JWT, encontrar a sessão local com aquele sid, e invalidá-la.

Configuração no Keycloak: Client / Settings / Backchannel Logout URL / https://minha-app.com/backchannel-logout

Front-Channel Logout

Alternativa ao backchannel: o Keycloak renderiza iframes ocultos apontando para a URL de logout de cada client. Menos confiável — bloqueado por ad blockers, políticas de privacidade do browser, e third-party cookie restrictions. Prefira backchannel.

Revogação de Refresh Token

No logout, revogue o refresh token para impedir que seja usado para obter novos access tokens:

POST /realms/meu-realm/protocol/openid-connect/revoke
Content-Type: application/x-www-form-urlencoded

token=refresh_token_value
&token_type_hint=refresh_token
&client_id=minha-app
&client_secret=secret

Session Timeout

⏱️ No Keycloak

SSO Session Idle: tempo sem atividade (padrão 30min)
SSO Session Max: tempo máximo total (padrão 10h)
Access Token Lifespan: vida do access token (padrão 5min)

⏱️ Na sua app

Idle timeout local (limpar sessão após inatividade)
Sincronizar com SSO Session Idle do KC
Não confiar apenas na expiração do token

Problemas Comuns

Sintoma Causa Solução
Logout em app A, mas app B continua logada Backchannel não configurado Configurar Backchannel Logout URL em cada client
Logout redireciona para página de erro post_logout_redirect_uri não registrada Adicionar URL em Client / Valid Post Logout Redirect URIs
Logout não funciona no SPA Só limpa state local, não chama Keycloak Usar auth.signoutRedirect() (não só limpar variáveis)
Usuário faz login de novo sem digitar senha Sessão SSO ainda ativa no Keycloak Implementar RP-Initiated Logout (não só local)

🎯 Key Takeaway

Logout correto exige coordenação entre sua app E o Keycloak. Sempre implemente RP-Initiated Logout (redirecionar ao KC) + Backchannel Logout (receber notificação) em produção. Revogue refresh tokens no logout.
