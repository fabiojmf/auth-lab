Node.js / Express

No ecossistema Node.js, você não precisa de frameworks pesados para integrar com Keycloak. Duas bibliotecas resolvem tudo: jose para validar tokens e openid-client para login flows.

🟢

Duas abordagens

Proteger API (Resource Server): valide JWT nas requisições. Login de usuários (Client): redirecione ao Keycloak e gerencie sessão.

Protegendo uma API (Resource Server)

Use a biblioteca jose — leve, sem dependências, mantida pela comunidade OIDC:

import { jwtVerify, createRemoteJWKSet } from 'jose';

const JWKS = createRemoteJWKSet(
  new URL('https://keycloak/realms/meu-realm/protocol/openid-connect/certs')
);

async function authMiddleware(req, res, next) {
  const token = req.headers.authorization?.replace('Bearer ', '');
  if (!token) return res.status(401).json({ error: 'No token' });

  try {
    const { payload } = await jwtVerify(token, JWKS, {
      issuer: 'https://keycloak/realms/meu-realm',
      audience: 'minha-api'
    });
    req.user = payload;
    next();
  } catch (e) {
    res.status(401).json({ error: 'Invalid token' });
  }
}

O createRemoteJWKSet busca e cacheia as chaves públicas automaticamente. Quando o Keycloak rotaciona chaves, a biblioteca busca as novas.

Autorização por Role

function requireRole(role) {
  return (req, res, next) => {
    const roles = req.user.realm_access?.roles || [];
    if (!roles.includes(role)) {
      return res.status(403).json({ error: 'Forbidden' });
    }
    next();
  };
}

// Uso:
app.get('/admin/users', authMiddleware, requireRole('admin'), (req, res) => {
  res.json({ users: ['...'] });
});

app.get('/profile', authMiddleware, (req, res) => {
  res.json({ sub: req.user.sub, email: req.user.email });
});

Login de Usuários (OAuth2 Client)

Para aplicações que precisam fazer login (não apenas validar tokens), use openid-client (v6+):

import * as client from 'openid-client';
import session from 'express-session';

// Discovery automático — retorna um Configuration
const config = await client.discovery(
  new URL('https://keycloak/realms/meu-realm'),
  'minha-app',
  'meu-secret'
);

const REDIRECT_URI = 'http://localhost:3000/callback';

// Rota de login
app.get('/login', async (req, res) => {
  const codeVerifier = client.randomPKCECodeVerifier();
  const codeChallenge = await client.calculatePKCECodeChallenge(codeVerifier);
  req.session.codeVerifier = codeVerifier;

  const authUrl = client.buildAuthorizationUrl(config, {
    redirect_uri: REDIRECT_URI,
    scope: 'openid profile email',
    code_challenge: codeChallenge,
    code_challenge_method: 'S256',
  });
  res.redirect(authUrl.href);
});

// Callback
app.get('/callback', async (req, res) => {
  const currentUrl = new URL(req.url, 'http://localhost:3000');
  const tokenResponse = await client.authorizationCodeGrant(config, currentUrl, {
    pkceCodeVerifier: req.session.codeVerifier,
  });
  req.session.tokens = tokenResponse;
  res.redirect('/');
});

Bibliotecas Recomendadas

Biblioteca Uso Quando usar
jose Validação de JWT APIs que só precisam validar tokens
openid-client (v6+) Client OIDC completo Apps que fazem login de usuários
passport-openidconnect Strategy para Passport.js Se já usa Passport no projeto

🎯 Key Takeaway

Para APIs, jose é tudo que você precisa — leve e sem dependências. Para login flows completos, use openid-client v6+ (API funcional, sem classes). Esqueça o keycloak-nodejs-connect (deprecated).
