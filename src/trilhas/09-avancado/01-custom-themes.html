Custom Themes

O Keycloak permite customizar completamente sua interface visual — login, console de conta, admin console e emails — sem alterar o código-fonte. Isso é feito através do sistema de themes.

🎨

O que são Themes?

Pacotes de templates, CSS, JS e imagens que definem a aparência de cada parte do Keycloak. Cada realm pode usar um theme diferente.

Tipos de Theme

TipoO que customizaExemplo
loginPágina de login, registro, reset de senhaLogo e cores da empresa
accountConsole de conta do usuárioBranding na área do usuário
adminAdmin ConsoleRaramente customizado
emailTemplates de email (verificação, reset)Email com identidade visual
welcomePágina inicial do KeycloakMensagem customizada

Estrutura de Diretórios

themes/ └── meu-tema/ └── login/ ├── theme.properties ├── resources/ │ ├── css/login.css │ └── img/logo.png └── templates/ └── login.ftl

O arquivo theme.properties

parent=keycloak import=common/keycloak styles=css/login.css scripts=js/custom.js

A propriedade parent=keycloak é a chave: herda tudo do tema padrão e você sobrescreve apenas o que precisa.

Extending vs Overriding

✅ Extending (recomendado)

parent=keycloak
Herda todos os templates e recursos
Sobrescreve apenas arquivos específicos
Atualizações do KC não quebram seu tema
Manutenção mínima entre versões

⚠️ Forking (evite)

Copiar todo o tema padrão
Modificar diretamente os templates
Sem herança — tudo é seu
Cada upgrade do KC pode quebrar tudo
Manutenção pesada e propensa a bugs

Deploy de Themes

MétodoComoQuando usar
DiretórioCopiar para /opt/keycloak/themes/Desenvolvimento local
JAR em providers/Empacotar como JAR + kc.sh buildContainers (recomendado)
Docker COPYCOPY meu-tema/ /opt/keycloak/themes/meu-tema/Imagem customizada
Volume mount-v ./themes:/opt/keycloak/themesDev com Docker

Estrutura do JAR (método recomendado para produção):

meu-tema.jar ├── META-INF/ │ └── keycloak-themes.json └── theme/ └── meu-tema/ └── login/ ├── theme.properties ├── resources/ └── templates/

O arquivo META-INF/keycloak-themes.json registra o tema:

{ "themes": [ { "name": "meu-tema", "types": ["login", "email"] } ] }

Após copiar o JAR para /opt/keycloak/providers/, execute kc.sh build para registrar o tema. Em Dockerfiles, isso é feito na etapa de build da imagem.

Customizações Comuns

Logo na página de login:

/* resources/css/login.css */ .login-pf-brand img { content: url('../img/minha-logo.svg'); max-height: 60px; }

Campos no registro: sobrescreva register.ftl adicionando campos com user attributes.

Emails customizados: crie templates .ftl em email/html/ e email/text/.

Cache de Themes

🏭 Produção

Themes são cacheados no startup
Alterações requerem restart
Performance otimizada

🧪 Desenvolvimento

Desabilitar cache para hot-reload:
--spi-theme-static-max-age=-1
--spi-theme-cache-themes=false

Atribuição de Themes

Por realm: Realm Settings / Themes tab — define o tema padrão para login, account e email.

Por client: Client / Settings / Login Theme — permite que um client específico use um tema diferente do realm. Útil para multi-branding.

🎯 Key Takeaway

Themes permitem personalizar o Keycloak visualmente sem tocar no código-fonte. Sempre use parent=keycloak para herdar o tema padrão e sobrescrever apenas o necessário. Nunca faça fork completo — a manutenção entre versões se torna insustentável.
