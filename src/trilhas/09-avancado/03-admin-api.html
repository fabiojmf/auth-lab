API Admin (REST)

O Keycloak expõe uma API REST completa para tudo que o Admin Console faz. Qualquer operação — criar clients, gerenciar usuários, configurar realms — pode ser automatizada via HTTP.

🤖

Por que usar a API?

Automação, CI/CD, scripts de migração, integração com ferramentas externas e dashboards customizados.

Autenticação na API

Primeiro, obtenha um token de admin:

curl -X POST \ https://keycloak.example.com/realms/master/protocol/openid-connect/token \ -d "client_id=admin-cli" \ -d "username=admin" \ -d "password=admin" \ -d "grant_type=password"

Ou use uma service account com roles de realm-management para evitar credenciais de usuário em scripts.

Base URL

https://keycloak.example.com/admin/realms/{realm}/

Operações Comuns

OperaçãoMétodoEndpoint
Listar clientsGET/admin/realms/{realm}/clients
Criar clientPOST/admin/realms/{realm}/clients
Buscar usuáriosGET/admin/realms/{realm}/users?search=maria
Criar usuárioPOST/admin/realms/{realm}/users
Definir senhaPUT/admin/realms/{realm}/users/{id}/reset-password
Atribuir rolePOST/admin/realms/{realm}/users/{id}/role-mappings/realm
Exportar realmGET/admin/realms/{realm}

Exemplo: Criar Usuário com Senha

# Criar usuário curl -X POST \ https://kc.example.com/admin/realms/my-app/users \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{ "username": "maria", "email": "maria@example.com", "enabled": true, "firstName": "Maria", "lastName": "Silva" }' # Definir senha (pegar ID do Location header ou buscar) curl -X PUT \ https://kc.example.com/admin/realms/my-app/users/$USER_ID/reset-password \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{ "type": "password", "value": "senha-segura-123", "temporary": false }'

Paginação

Endpoints que retornam listas suportam paginação:

GET /admin/realms/my-app/users?first=0&max=100

O padrão é max=100. Para buscar todos, itere incrementando first até receber lista vazia.

Tratamento de Erros

StatusCausaSolução
401Token expirado ou inválidoRenovar token antes de cada batch
403Permissões insuficientesVerificar roles da service account
409Conflito (recurso já existe)Verificar existência antes de criar
404Recurso não encontradoVerificar realm/ID corretos

Casos de Uso Reais

  • CI/CD: provisionar clients automaticamente no deploy
  • Migração: scripts para importar usuários de outro sistema
  • Bulk ops: desabilitar usuários inativos, atribuir roles em massa
  • Dashboards: painel customizado com métricas de usuários/sessões

🎯 Key Takeaway

Tudo que você faz no Admin Console pode ser automatizado via API REST. Use service accounts com roles mínimas, trate erros de token expirado, e pagine resultados. É a base para qualquer automação séria com Keycloak.
