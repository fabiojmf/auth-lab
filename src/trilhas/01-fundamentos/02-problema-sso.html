O problema que o SSO resolve

Antes de entender o SSO, veja o problema que ele resolve. Alterne entre os dois cenários:

👤
Maria
📧 Email maria@email / senha123
📊 ERP maria.silva / Erp@2024
📁 Documentos m.silva / Doc#456!
💬 Chat maria_s / Chat789
  • Múltiplas senhas para lembrar
  • Cada sistema tem sua base de usuários
  • Demitiu? Revogar acesso em N sistemas
  • Sem padrão de segurança
👤
Maria
🔐 SSO (Keycloak) maria@empresa / S3nh@F0rte!
📧 Email
📊 ERP
📁 Docs
💬 Chat
  • Uma senha só, forte e bem protegida
  • Base de usuários centralizada
  • Demitiu? Revoga em um lugar, bloqueia tudo
  • Segurança padronizada (MFA, políticas)

O que é SSO, afinal?

Single Sign-On = um login, acesso a tudo. O usuário se autentica uma vez no servidor central (o SSO) e ganha acesso a todos os sistemas conectados sem precisar logar novamente.

Como funciona na prática

👤

1. Usuário acessa o sistema de Documentos

↗️

2. Sistema redireciona para o SSO (Keycloak)

🔐

3. Usuário digita login e senha no SSO

✅

4. SSO confirma: "É a Maria". Redireciona de volta.

📁

5. Sistema de Documentos recebe a confirmação. Maria está logada.

📊

6. Maria acessa o ERP. O SSO já tem uma sessão ativa dela — entra direto, sem logar de novo.

Isso é o "Single" do Single Sign-On: uma sessão no SSO vale para todos os sistemas conectados.

Onde o Keycloak entra

Keycloak é um servidor SSO open source. No fluxo acima, ele é quem faz os passos 2, 3 e 4 — recebe o redirect, mostra a tela de login, valida as credenciais e devolve a confirmação. Além disso, ele:

  • Gerencia usuários e suas credenciais em um lugar só
  • Mantém a sessão (por isso o segundo sistema não pede login de novo)
  • Emite tokens — a "prova" de que o usuário logou, que os sistemas confiam
  • Conecta com bases externas (LDAP, Active Directory) para reaproveitar usuários que já existem
  • Suporta MFA, políticas de senha, brute force protection

No Keycloak, tudo é organizado em realms — espaços isolados que contêm seus próprios usuários, clients e configurações. Pense em um realm como um inquilino (tenant). Cada realm é independente dos demais.

💡 Ponto-chave

SSO não é um protocolo — é um conceito. OAuth2 e OIDC são os protocolos que fazem o SSO funcionar. Keycloak é uma implementação desses protocolos.
