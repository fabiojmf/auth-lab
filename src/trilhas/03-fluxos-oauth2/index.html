Fluxos OAuth2/OIDC

Como a autenticação realmente acontece. Cada fluxo animado passo a passo — veja os requests, redirects e tokens sendo trocados.

1

Authorization Code Flow

~10 min

O fluxo padrão. Usuário loga via redirect, aplicação recebe um code e troca por token. O mais seguro e mais usado.
2

Client Credentials

~6 min

Máquina falando com máquina. Sem usuário, sem browser. O client se autentica como ele mesmo.
3

Direct Grant (Resource Owner Password)

A aplicação recebe a senha do usuário e troca por token. Legado, mas ainda usado.
4

Refresh Token

~7 min

Como renovar o acesso sem pedir login de novo. O ciclo de vida dos tokens.
5

Quando usar cada fluxo

~7 min

Quiz interativo: dado um cenário, qual fluxo escolher? Árvore de decisão prática.
