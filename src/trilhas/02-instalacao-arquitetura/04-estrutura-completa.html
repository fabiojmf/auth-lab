A Estrutura Completa: Clients, Users, Roles, Groups

Quatro entidades formam o coração do Keycloak. Entenda como elas se relacionam e você entende o modelo inteiro.

As 4 entidades principais

👤 Users
📱 Clients
🎭 Roles
👥 Groups

Pessoas (ou service accounts) que se autenticam no Keycloak. Cada user tem username, email, credenciais e pode pertencer a groups e ter roles.

Relações: pertence a Groups, tem Roles (direto ou via Group), acessa Clients

Como se relacionam

👥 Groups
👤 Users
🎭 Roles
📱 Clients
👥 Groups contêm 👤 Users
👥 Groups têm 🎭 Roles
👤 Users herdam 🎭 Roles via Group
👤 Users podem ter 🎭 Roles direto
📱 Clients definem 🎭 Client Roles

Realm Roles vs Client Roles

🌐 Realm Roles

  • Globais ao realm
  • Ex: "admin", "user", "manager"
  • No token: realm_access.roles
  • Qualquer client pode verificar

📱 Client Roles

  • Específicas de um client
  • Ex: "editor" no app X
  • No token: resource_access.{client}.roles
  • Escopo mais restrito

🎫 Como fica no token

{
  "realm_access": {
    "roles": ["user", "manager"]
  },
  "resource_access": {
    "my-app": {
      "roles": ["editor", "viewer"]
    },
    "another-app": {
      "roles": ["admin"]
    }
  }
}

Criando cada entidade

  1. Menu lateral / Users
  2. Clique Add User
  3. Preencha: username, email, first/last name
  4. Salve / vá na aba Credentials
  5. Set Password / defina uma senha / Temporary: OFF
  1. Menu lateral / Clients
  2. Clique Create Client
  3. Client ID: nome único (ex: my-app)
  4. Client type: OpenID Connect
  5. Next / configure Client authentication (ON/OFF) e redirect URIs
  1. Menu lateral / Realm Roles
  2. Clique Create Role
  3. Role name: nome descritivo (ex: can-edit)
  4. Opcionalmente adicione uma descrição
  5. Salve. Agora atribua a users ou groups.
  1. Menu lateral / Groups
  2. Clique Create Group
  3. Name: nome do grupo (ex: developers)
  4. Salve / aba Role Mapping / atribua roles
  5. Aba Members / adicione usuários
