Subindo o Keycloak

Chega de teoria. Neste módulo você vai colocar um Keycloak real para rodar na sua máquina — em menos de 5 minutos.

Pré-requisitos

🐳 Docker

  • Docker Engine 20.10+
  • Instalar Docker
  • Verifique: docker --version

🧩 Docker Compose

  • Compose V2 (integrado ao Docker)
  • Instalar Compose
  • Verifique: docker compose version

O Docker Compose

O AuthLab já vem com um docker-compose.yml pronto. Ele sobe dois containers:

ServiçoImagemPortaFunção
keycloakquay.io/keycloak/keycloak:26.2.68080Servidor de identidade
postgrespostgres:18.4-alpine5432Banco de dados

O arquivo completo (docker/docker-compose.yml):

services:
  keycloak:
    image: quay.io/keycloak/keycloak:26.2.6
    command: start-dev --import-realm
    ports:
      - "8080:8080"
    environment:
      KEYCLOAK_ADMIN: admin
      KEYCLOAK_ADMIN_PASSWORD: admin
      KC_DB: postgres
      KC_DB_URL: jdbc:postgresql://postgres:5432/keycloak
      KC_DB_USERNAME: keycloak
      KC_DB_PASSWORD: keycloak
    volumes:
      - ./realm-export:/opt/keycloak/data/import
    depends_on:
      postgres:
        condition: service_healthy

  postgres:
    image: postgres:18.4-alpine
    ports:
      - "5432:5432"
    environment:
      POSTGRES_DB: keycloak
      POSTGRES_USER: keycloak
      POSTGRES_PASSWORD: keycloak
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U keycloak"]
      interval: 5s
      timeout: 3s
      retries: 5

volumes:
  pgdata:

Pontos importantes:

  • start-dev — modo desenvolvimento (sem HTTPS obrigatório)
  • --import-realm — importa configurações prontas ao iniciar
  • depends_on + healthcheck — Keycloak só sobe quando o Postgres estiver pronto
  • KEYCLOAK_ADMIN — cria o usuário admin no primeiro boot

Passo a passo

1
Crie uma pasta e o arquivo 
mkdir keycloak-lab && cd keycloak-lab

Crie um arquivo docker-compose.yml nessa pasta com o conteúdo mostrado acima.

2
Suba os containers 
docker compose up -d

O -d roda em background. Primeira vez demora mais (download das imagens).

3
Acompanhe os logs 
docker compose logs -f keycloak

Espere até ver: Keycloak ... started in Xs. Ctrl+C para sair dos logs.

4
Acesse o console

Abra no navegador: http://localhost:8080

Clique em "Administration Console".

5
Faça login

Usuário: admin  |  Senha: admin

Pronto! Você está no Admin Console.

O que você está vendo

O Admin Console é, ele mesmo, um client do Keycloak — uma aplicação web (React) que se autentica via OpenID Connect no próprio servidor. Tudo que você faz nele é uma chamada à Admin REST API.

Isso significa: tudo que o console faz, você pode fazer via API. O console é só uma interface gráfica.

Troubleshooting

⚠️ Porta 8080 já em uso +

Outro serviço está usando a porta. Edite o docker-compose.yml:

ports:
  - "8180:8080"  # acesse em localhost:8180

⚠️ Container reiniciando em loop +

Geralmente é problema de conexão com o banco. Verifique:

docker compose logs postgres
docker compose ps

Se o Postgres não está healthy: docker compose down -v && docker compose up -d

⚠️ Startup muito lento +

Normal na primeira execução! O Keycloak cria todo o schema do banco (dezenas de tabelas). Pode levar 30-60s. Nas próximas vezes será instantâneo.
