Configuração de um Client

Ao criar um client no Keycloak, você encontra vários campos. Cada um tem um propósito específico. Clique em qualquer campo abaixo para ver a explicação detalhada.

Tabela de referência rápida

Campo O que faz Valor típico
Client ID Identificador único do client minha-app-backend
Client authentication On = confidential, Off = public On (backend) / Off (SPA)
Standard Flow Habilita Authorization Code Flow On (se tem login humano)
Direct Access Grants Habilita Resource Owner Password Off (exceto CLI/legado)
Service accounts Habilita Client Credentials On (se máquina-a-máquina)
Root URL Base para URLs relativas https://app.com
Valid Redirect URIs URIs permitidas para redirect após login https://app.com/callback
Web Origins Origens permitidas para CORS + (usa redirect URIs)
Flow Overrides Usa um flow de autenticação diferente (vazio = usa o padrão do realm)

💡 Dicas importantes

Redirect URIs: nunca use * em produção. Seja o mais específico possível.
Web Origins: use + para derivar automaticamente das redirect URIs.
Direct Access Grants: desabilite a menos que realmente precise (é menos seguro).
Service accounts: só funciona com Client authentication: On.
