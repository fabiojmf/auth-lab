Monitoramento e Métricas

Você não pode corrigir o que não consegue ver. Monitoramento é o que separa "o Keycloak caiu e ninguém sabe por quê" de "detectamos o problema antes dos usuários reclamarem".

📊

Observabilidade desde o dia 1

Habilite health checks e métricas antes de ir para produção. Não depois do primeiro incidente.

Health Endpoints

O Keycloak expõe endpoints de saúde prontos para uso com orquestradores (Kubernetes, ECS, etc.):

EndpointPergunta que respondeUso típico
/health/readyKC está pronto para receber requests?Readiness probe (K8s)
/health/liveO processo está vivo?Liveness probe (K8s)
/health/startedO startup terminou?Startup probe (K8s)
# Habilitar health endpoints kc.sh start --health-enabled=true # Testar (KC 20-24: porta 8080, KC 25+: porta 9000 — management interface separada) curl http://localhost:9000/health/ready # {"status": "UP", "checks": [{"name": "Database", "status": "UP"}]} # Se estiver em KC 20-24 (antes da management interface): # curl http://localhost:8080/health/ready

Métricas (Prometheus)

Com --metrics-enabled=true, o Keycloak expõe métricas no formato Prometheus/OpenMetrics em /metrics (porta 9000 no KC 25+, porta 8080 no KC 20-24). As métricas de eventos (logins, erros) requerem que o event listener metrics-listener esteja habilitado no realm:

MétricaTipoO que indica
keycloak_loginsCounterTotal de logins (por realm, client)
keycloak_login_errorsCounterFalhas de login (por tipo de erro)
keycloak_registrationsCounterNovos registros de usuários
keycloak_refresh_tokensCounterTokens renovados
jvm_memory_used_bytesGaugeUso de memória da JVM
jvm_gc_pause_secondsSummaryPausas do Garbage Collector
http_server_requests_secondsHistogramLatência de requests HTTP
agroal_active_countGaugeConexões ativas no pool do DB
# Habilitar métricas kc.sh start --metrics-enabled=true # Coletar métricas (KC 25+: porta 9000; KC 20-24: porta 8080) curl http://localhost:9000/metrics # Para métricas de eventos (logins, erros), habilite no realm: # Admin Console / Realm Settings / Events / Event Listeners / adicionar "metrics-listener" # # keycloak_logins_total{realm="meu-realm",client_id="app-web"} 1523 # keycloak_login_errors_total{realm="meu-realm",error="invalid_user_credentials"} 47

Alertas Recomendados

🚨 Alertas que você deve configurar

• Login error rate spike: aumento súbito em keycloak_login_errors / possível ataque

• Health endpoint down: /health/ready retornando não-200 / instância com problema

• DB pool exhausted: agroal_active_count = max / queries lentas ou leak

• High GC pause: jvm_gc_pause_seconds > 1s / precisa de mais memória

• HTTP latency P99 > 2s: degradação de performance

🎯 Key Takeaway

Habilite --health-enabled=true e --metrics-enabled=true desde o dia 1. Você não pode corrigir o que não consegue ver. Grafana + Prometheus é o stack padrão para métricas do Keycloak.
