Alta Disponibilidade (Clustering)

O Keycloak é o ponto único de autenticação de todos os sistemas conectados. Se ele cai, ninguém faz login em nada. Alta disponibilidade não é luxo — é requisito.

🔗

SSO = Single Point of Failure

Se o Keycloak autentica 20 sistemas e fica indisponível, 20 sistemas perdem autenticação simultaneamente.

Como o Keycloak forma cluster

O clustering do Keycloak é baseado em três pilares:

1
Banco de dados compartilhado

Todas as instâncias leem/escrevem no mesmo banco. Realms, users, clients — tudo está lá.

2
Cache distribuído (Infinispan)

Sessões, tokens ativos e login failures são compartilhados via cache in-memory entre nós.

3
Sticky sessions opcionais (mas recomendadas)

Qualquer instância pode atender qualquer request graças ao cache distribuído. Porém, sticky sessions melhoram performance ao evitar lookups remotos no Infinispan — especialmente durante fluxos de login (auth sessions). Use se possível, mas não é obrigatório.

Camadas de Cache

📦 Cache Local

Por instância (não compartilhado)
Realm metadata, client configs
Invalidado via mensagens do cluster
Reduz queries ao banco

🌐 Cache Distribuído

Compartilhado entre todos os nós
User sessions, auth sessions
Login failures, action tokens
Sincronizado via JGroups (TCP/UDP)

Load Balancer

⚖️ Requisitos do Load Balancer

• Sticky sessions: Opcionais mas recomendadas para performance (cookie KC_AUTH_SESSION_ID)

• Health check: GET /health/ready (HTTP 200 = saudável)

• WebSocket: necessário para o Admin Console funcionar

• Algoritmo: round-robin ou least-connections

Rolling Update

Durante um rolling update em Kubernetes:

1
Novo pod sobe

Junta-se ao cluster Infinispan, recebe dados do cache distribuído.

2
Pod antigo é drenado

Para de receber requests, mas sessões já estão no cache compartilhado.

3
Usuários não percebem

Sessões sobrevivem porque estão no cache distribuído, não na memória local.

🎯 Key Takeaway

Keycloak clusteriza via banco compartilhado + cache distribuído (Infinispan). Sticky sessions são opcionais mas melhoram performance. Em Kubernetes, use Deployment + --cache-stack=kubernetes e pronto.
