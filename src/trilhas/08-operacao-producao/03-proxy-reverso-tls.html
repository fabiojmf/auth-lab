Proxy Reverso e TLS

Em produção, o Keycloak nunca fica exposto diretamente à internet. Um reverse proxy na frente cuida de TLS termination, load balancing, WAF e rate limiting.

🔒

Arquitetura padrão

Cliente / HTTPS / Reverse Proxy / HTTP / Keycloak (rede interna)

Proxy Headers no Keycloak 20+

Quando o Keycloak está atrás de um proxy, ele precisa saber o protocolo e IP originais do cliente. Sem essa configuração, ele gera URLs erradas (HTTP ao invés de HTTPS).

📋 xforwarded

--proxy-headers=xforwarded
Confia em X-Forwarded-For
Confia em X-Forwarded-Proto
Confia em X-Forwarded-Port
Padrão de facto (NGINX, HAProxy, ALB)

📋 forwarded

--proxy-headers=forwarded
Confia no header Forwarded (RFC 7239)
Header único e padronizado
Formato: for=ip;proto=https;host=sso.com
Menos comum na prática
⚠️

Sem --proxy-headers: o Keycloak vê o IP interno do proxy e acha que está em HTTP. Resultado: URLs de redirect quebradas, mixed content errors, loops infinitos.

⚠️

HTTP no Keycloak: Se o proxy termina TLS e fala HTTP com o Keycloak, você precisa de --http-enabled=true no KC (pois o modo produção desabilita HTTP por padrão). Combine com --proxy-headers=xforwarded para que o KC saiba que o cliente original usou HTTPS.

Configuração NGINX

upstream keycloak { server kc1:8080; server kc2:8080; } server { listen 443 ssl; server_name sso.empresa.com; ssl_certificate /etc/ssl/certs/sso.empresa.com.pem; ssl_certificate_key /etc/ssl/private/sso.empresa.com.key; location / { proxy_pass http://keycloak; proxy_set_header Host $host; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; proxy_set_header X-Forwarded-Port $server_port; # Buffers grandes (SAML tokens podem ser enormes) proxy_buffer_size 128k; proxy_buffers 4 256k; # WebSocket (Admin Console) proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade"; } }

Problemas Comuns

SintomaCausaSolução
"Mixed content" no browser KC gera URLs HTTP, mas página é HTTPS Configurar --proxy-headers=xforwarded
Redirect loop infinito X-Forwarded-Proto não está sendo enviado Adicionar proxy_set_header X-Forwarded-Proto $scheme
502 Bad Gateway em login SAML Headers/cookies muito grandes para o buffer Aumentar proxy_buffer_size 128k
Admin Console não carrega WebSocket bloqueado pelo proxy Adicionar headers Upgrade + Connection

🎯 Key Takeaway

Sempre rode Keycloak atrás de um reverse proxy em produção. Configure --proxy-headers=xforwarded ou as URLs geradas pelo Keycloak estarão erradas — causando mixed content, redirect loops e frustração.
