Arquitetura e Deploy

O Keycloak moderno (20+) roda sobre Quarkus — um runtime Java otimizado para containers e cloud. Isso mudou completamente como se configura e deploya o Keycloak em produção.

🏗️

Quarkus vs WildFly (Legacy)

Antes do KC 17, o Keycloak rodava sobre WildFly (JBoss). A migração para Quarkus trouxe startup mais rápido, menor consumo de memória e configuração simplificada.

Modos de Deploy

O Keycloak tem dois modos fundamentais — e confundir os dois é o erro #1 de quem vai para produção:

🧪 Development

kc.sh start-dev
Banco H2 embutido (dados em memória)
HTTP habilitado (sem HTTPS)
Sem otimização de build
Hostname não validado
⚠️ Nunca use em produção

🏭 Production

kc.sh start
Banco externo obrigatório (PostgreSQL, MySQL...)
HTTPS obrigatório
Requer kc.sh build prévio
Hostname configurado e validado
✅ Otimizado para performance

O passo build

O comando kc.sh build pré-compila a configuração (providers, banco, features) em artefatos otimizados. Isso elimina processamento no startup e reduz o tempo de inicialização drasticamente.

# Build: compila configuração uma vez /opt/keycloak/bin/kc.sh build --db=postgres --features=token-exchange # Start: usa o build pré-compilado /opt/keycloak/bin/kc.sh start --hostname=sso.empresa.com --db-url=jdbc:postgresql://db:5432/keycloak

Connection pool: Configure --db-pool-initial-size (default: 0) e --db-pool-max-size (default: 100). O schema é migrado automaticamente no startup (via Liquibase) — o Keycloak cria/atualiza tabelas sozinho. Nunca altere o schema manualmente.

Deploy com Container

A imagem oficial é quay.io/keycloak/keycloak. O padrão é um Dockerfile multi-stage:

FROM quay.io/keycloak/keycloak:latest as builder ENV KC_DB=postgres ENV KC_FEATURES=token-exchange RUN /opt/keycloak/bin/kc.sh build FROM quay.io/keycloak/keycloak:latest COPY --from=builder /opt/keycloak/ /opt/keycloak/ # --optimized pula o build step no startup (requer build prévio no stage anterior) ENTRYPOINT ["/opt/keycloak/bin/kc.sh"] CMD ["start", "--optimized"]

🎯 Key Takeaway

Keycloak em produção requer configuração explícita — nada funciona por padrão. Banco externo, HTTPS, hostname e proxy-headers são o mínimo absoluto. Sem isso, você tem um ambiente de desenvolvimento exposto na internet.
